⚠️ SSO Environment — Why Basic Auth Doesn't Work:
Your company uses Microsoft Entra ID (Azure AD) SSO for SNOW login. This means your normal username/password can't be used in Power Automate directly — SSO accounts don't have a standalone password that connectors can use. You have two options below.
✅ Recommended — Path A

Ask SNOW Admin for a Service Account

Request a local (non-SSO) service account from your SNOW admin. This is a standard practice and takes 1–2 days. You get full read+write access to Power Automate.

✅ Full automation possible
✅ Can auto-update tickets
✅ Reliable & stable
⏱ Needs admin approval
🔄 Alternative — Path B

Power Automate Desktop (RPA)

No admin needed. Installs on your PC, opens SNOW in the browser like a robot, reads and updates tickets using your SSO login automatically.

✅ No admin approval needed
✅ Uses your SSO login
⚠️ PC must be ON and logged in
⚠️ Slightly slower setup
A
Path A — Request a Service Account from SNOW Admin
Send this email to your SNOW admin → get credentials → connect Power Automate
1
Send this email to your SNOW Admin
Copy and send this exact email. Keep it simple — this is a very standard request.
To: your-snow-admin@company.com
Subject: Request: Local Service Account for Power Automate Integration
Hi [Admin Name],

I'd like to set up a Power Automate flow to monitor my assigned ServiceNow incidents (P1/P2 alerts). Since our SNOW uses SSO, I need a local service account with Basic Authentication to use with the Power Automate ServiceNow connector.

Could you please create a local SNOW user account with:
• Role: itil (read incidents + add work notes)
• Username: svc_powerautomate_[myname] (or any naming you prefer)
• Tables needed: incident (read), incident (update work_notes field only)

This account will only be used by my personal Power Automate flow to read my assigned tickets and post acknowledgement notes. No admin access needed.

Please share the username and password securely. Thank you!

Regards,
[Your Name]
✅ Why this works: SNOW supports both SSO (for human login) and Basic Auth (for integrations) simultaneously. A local account bypasses SSO entirely.

4
Auto-Update SNOW Ticket (Work Note Acknowledgement)
Automatically write "Acknowledged" in the ticket when it's first assigned to you
⚠️ Auto-update requires Path A (service account with write access). Path B (RPA) can also do this but requires more recording steps. The service account needs the itil role at minimum to write work notes.
1
Add "Apply to each" loop — loop through each P1 ticket
Inside the YES branch, after the email step → + Add action → search "Apply to each" (Control). This loops through every P1 ticket found.
Control — Loop
🔁 Apply to each — P1 tickets
Select output from:body('List_Records_P1')?['result']
👆 Click "Select an output from previous steps" and pick result from List Records P1
2
Inside the loop: Update Record in SNOW
Inside the Apply to each loop → + Add action → ServiceNow → "Update Record"
Action — ServiceNow
✍️ Update Record — Add Work Note
Connection:ServiceNow (yourcompany) ✓
Table Name:Incident [incident]
System ID:items('Apply_to_each')?['sys_id'] ← dynamic
work_notes:Acknowledged by [Your Name] via Power Automate. Will review and update shortly. - Auto message @{utcNow()}
💡 System ID field: Click "Add dynamic content" → find sys_id from the Apply to each loop. This is the unique ID of each ticket — Power Automate uses it to know which ticket to update.
✅ Result: Each P1/P2 ticket assigned to you will automatically get a work note like: "Acknowledged by John Smith via Power Automate. Will review and update shortly. - Auto message 2026-04-09T08:30:00Z"
3
Repeat the loop for P2 tickets
Add another Apply to each loop below the first one, this time using body('List_Records_P2')?['result'] as input. Paste the same Update Record action inside it.
⚠️ Avoid duplicate notes: The flow runs every 15 minutes. If a ticket stays assigned for hours, it will get many "Acknowledged" notes. To prevent this, add a condition inside the loop: only update if work_notes does NOT already contain "Acknowledged". Ask Claude to help write that condition if needed!

priority=2 → P2 High
