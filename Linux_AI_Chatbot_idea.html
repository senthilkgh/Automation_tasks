Step-by-Step Installation
On your dedicated Linux server (Ubuntu 22.04/24.04 or RHEL/Rocky)
1
Download Ollama on an internet-connected machine (one time)
Since your target server has no internet, download on a machine that does, then transfer via USB or internal network.
# On internet machine — download Ollama installer
curl -fsSL https://ollama.com/install.sh -o ollama-install.sh

# Download the model file (e.g. Llama 3.3 8B = ~5GB)
ollama pull llama3.3:8b
# Models saved at: ~/.ollama/models/

# Copy both to USB drive or internal NFS share
cp -r ~/.ollama/models /media/usb/ollama-models/
cp ollama-install.sh /media/usb/
2
Install Ollama on the air-gapped server (offline)
# On your air-gapped Linux server — plug in USB
sudo mount /dev/sdb1 /mnt/usb

# Run Ollama installer (works offline — no download needed)
sudo sh /mnt/usb/ollama-install.sh

# Copy models to Ollama models directory
cp -r /mnt/usb/ollama-models/* ~/.ollama/models/

# Verify models are detected
ollama list
# Should show: llama3.3:8b

# Test it works
ollama run llama3.3:8b "What is the command to check disk usage?"
3
Configure Ollama as a systemd service (auto-start on boot)
# Make Ollama listen on all interfaces so team can connect
sudo nano /etc/systemd/system/ollama.service

# Add/modify these lines inside [Service]:
Environment="OLLAMA_HOST=0.0.0.0:11434"
Environment="OLLAMA_NUM_PARALLEL=4"
Environment="OLLAMA_MAX_LOADED_MODELS=2"

sudo systemctl daemon-reload
sudo systemctl enable ollama
sudo systemctl start ollama
sudo systemctl status ollama
4
Install Open WebUI (offline via Docker or pip)
# Option A: Docker (pre-pull image on internet machine, transfer tar)
# On internet machine:
docker pull ghcr.io/open-webui/open-webui:main
docker save ghcr.io/open-webui/open-webui:main -o open-webui.tar
# Copy open-webui.tar to USB → transfer to air-gapped server

# On air-gapped server:
docker load -i /mnt/usb/open-webui.tar
docker run -d \
  --name open-webui \
  -p 3000:8080 \
  -v open-webui:/app/backend/data \
  -e OLLAMA_BASE_URL=http://localhost:11434 \
  ghcr.io/open-webui/open-webui:main

# Access from any PC on your network:
# http://your-server-ip:3000
5
Set up custom Linux Admin system prompt in Open WebUI
In Open WebUI → Settings → Admin Panel → System Prompt — paste this to make the AI behave as a Linux sysadmin expert:
# System Prompt — paste in Open WebUI Admin → System Prompt

You are an expert Linux system administrator with 15+ years of experience.
You work in an air-gapped, no-internet environment.

When asked for commands:
- Always provide the exact command with explanation
- Add warnings for destructive operations (rm -rf, etc.)
- Show both the command AND expected output
- Prefer RHEL/CentOS/Rocky syntax when distro is unspecified
- For scripts, include error handling and logging

Topics you handle:
- systemd service management
- Disk/filesystem troubleshooting
- Network diagnostics (no internet)
- Log analysis (journalctl, /var/log)
- User/permission management
- Cron jobs and automation
- Performance monitoring (top, vmstat, iostat)
- Package management (yum/dnf/apt offline)
6
Verify everything works — test from a team member's PC
# From any PC on the internal network, open browser:
http://192.168.x.x:3000  ← replace with your server IP

# Or test via curl from terminal:
curl http://192.168.x.x:11434/api/generate \
  -d '{"model":"llama3.3:8b","prompt":"How do I check disk space in Linux?"}'
✅ If you get a JSON response with Linux commands — your AI is fully operational offline!

Offline Update Procedures
How to update AI models and software without internet — USB or LAN transfer
1
Update Method 1 — USB Drive Transfer (simplest)
### ON INTERNET MACHINE (outside your network) ###

# Pull new/updated model
ollama pull llama3.3:8b-instruct-q8_0 # newer version

# Find where model is stored
ls ~/.ollama/models/blobs/

# Copy ENTIRE .ollama directory to USB
rsync -av ~/.ollama/models/ /media/usb/ollama-models/

### ON AIR-GAPPED SERVER ###

sudo mount /dev/sdb1 /mnt/usb
rsync -av /mnt/usb/ollama-models/ ~/.ollama/models/
ollama list # verify new model appears
sudo systemctl restart ollama
2
Update Method 2 — Internal NFS / SFTP Server (best for teams)
Set up a dedicated "update server" on your internal network that has occasional internet access. Admins update it, then all AI servers pull from it internally.
# On internal UPDATE SERVER — sync models from internet
ollama pull llama3.3:8b
cp -r ~/.ollama/models/ /srv/nfs/ai-models/

# On AIR-GAPPED AI SERVER — mount internal NFS
sudo mount update-server:/srv/nfs/ai-models /mnt/ai-update
rsync -av /mnt/ai-update/ ~/.ollama/models/
sudo systemctl restart ollama

# Automate with a cron job — runs at 2am on Saturdays
0 2 * * 6 rsync -av update-server:/srv/nfs/ai-models/ ~/.ollama/models/
3
Update Open WebUI (Docker image)
# On internet machine — pull new image and export
docker pull ghcr.io/open-webui/open-webui:main
docker save ghcr.io/open-webui/open-webui:main | gzip > open-webui-latest.tar.gz
# Transfer to USB/NFS share

# On air-gapped server
docker stop open-webui
docker load -i /mnt/usb/open-webui-latest.tar.gz
docker start open-webui
# Data/chat history preserved in Docker volume automatically

Security Considerations
Running AI on internal infra — things to lock down
RiskMitigationPriority
Unauthorized team access to AIEnable Open WebUI authentication — admin creates user accountsHigh
AI suggests dangerous commandsSystem prompt includes warnings; never run AI output blindly — review firstHigh
Sensitive data in chat logsOpen WebUI stores chats locally — encrypt the disk, restrict accessMedium
Ollama API exposed internallyFirewall port 11434 to only AI server — users access via WebUI onlyMedium
Malicious USB/model injectionOnly use models from official Ollama library — verify checksumsMedium
USB drive policyOnly designated admin USB allowed — scan before plugging inLow
# Firewall rules — only allow WebUI port, block direct Ollama API
sudo firewall-cmd --permanent --add-port=3000/tcp # WebUI
sudo firewall-cmd --permanent --remove-port=11434/tcp # Block direct API
sudo firewall-cmd --reload

💻 Bonus: Terminal AI Wrapper Script
Save as /usr/local/bin/ai — lets any admin ask AI from terminal with: ai "how do I check open ports?"
#!/bin/bash
# /usr/local/bin/ai — Linux Admin AI Assistant
# Usage: ai "your question here"

if [ -z "$1" ]; then
  echo "Usage: ai 'your linux question'"
  exit 1
fi

MODEL="llama3.3:8b"
OLLAMA_URL="http://localhost:11434"

PROMPT="You are a Linux sysadmin expert. Give exact commands with brief explanation. Question: $1"

curl -s $OLLAMA_URL/api/generate \
  -d "{\"model\":\"$MODEL\",\"prompt\":\"$PROMPT\",\"stream\":false}" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['response'])"

# Make executable: chmod +x /usr/local/bin/ai
# Now use: ai "how do I check memory usage?"
