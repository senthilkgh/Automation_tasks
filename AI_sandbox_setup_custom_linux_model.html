Phase 1 — Install Ollama
Internet machine only — then transfer offline. Takes 5 minutes.
1
Method A — Direct install (if sandbox has internet temporarily)
bash — run on sandbox server
# One-line install — detects OS automatically curl -fsSL https://ollama.com/install.sh | sh # Verify install ollama --version # Expected: ollama version 0.6.x or higher # Check service status sudo systemctl status ollama
2
Method B — Offline install (no internet on sandbox)
bash — run on INTERNET machine first
### STEP 1: On internet machine — download everything ### mkdir -p ~/ollama-offline-bundle # Download install script curl -fsSL https://ollama.com/install.sh \ -o ~/ollama-offline-bundle/install.sh # Download Ollama binary directly (Linux x86_64) curl -L https://github.com/ollama/ollama/releases/latest/download/ollama-linux-amd64 \ -o ~/ollama-offline-bundle/ollama-binary # Make executable chmod +x ~/ollama-offline-bundle/ollama-binary # Copy bundle to USB drive cp -r ~/ollama-offline-bundle/ /media/usb/ echo "Bundle ready on USB!"
bash — run on SANDBOX (air-gapped)
### STEP 2: On sandbox server — install from USB ### sudo mount /dev/sdb1 /mnt/usb # adjust sdb1 to your USB device # Install Ollama binary manually sudo cp /mnt/usb/ollama-offline-bundle/ollama-binary /usr/local/bin/ollama sudo chmod +x /usr/local/bin/ollama # Create ollama user and group sudo useradd -r -s /bin/false -U -m -d /usr/share/ollama ollama sudo usermod -a -G ollama $USER # Create systemd service file sudo tee /etc/systemd/system/ollama.service <<EOF [Unit] Description=Ollama Service After=network-online.target [Service] ExecStart=/usr/local/bin/ollama serve User=ollama Group=ollama Restart=always RestartSec=3 Environment="HOME=/usr/share/ollama" Environment="OLLAMA_HOST=0.0.0.0:11434" Environment="OLLAMA_NUM_PARALLEL=2" Environment="OLLAMA_MAX_LOADED_MODELS=1" [Install] WantedBy=default.target EOF sudo systemctl daemon-reload sudo systemctl enable ollama sudo systemctl start ollama sudo systemctl status ollama
3
Configure Ollama for minimal RAM usage
bash — tune for low RAM
# Edit Ollama service for minimal resource use sudo nano /etc/systemd/system/ollama.service # Add these lines under [Service] section: Environment="OLLAMA_NUM_PARALLEL=1" # 1 request at a time (saves RAM) Environment="OLLAMA_MAX_LOADED_MODELS=1" # only 1 model in memory Environment="OLLAMA_NUM_THREAD=4" # match your CPU cores Environment="OLLAMA_KEEP_ALIVE=5m" # unload model after 5min idle Environment="OLLAMA_HOST=0.0.0.0:11434" # listen on all interfaces sudo systemctl daemon-reload sudo systemctl restart ollama
⚠️ OLLAMA_NUM_THREAD — set this to your actual CPU core count. Check with: nproc. Using all cores speeds up inference significantly on CPU-only.

Phase 3 — Install Open WebUI (Browser Chat Interface)
So your team can use it from any PC on the internal network
1
Option A — Docker (recommended, easiest)
bash — on INTERNET machine: pull & export Docker image
# Pull Open WebUI Docker image docker pull ghcr.io/open-webui/open-webui:main # Export to a tar file for offline transfer docker save ghcr.io/open-webui/open-webui:main \ | gzip > ~/open-webui.tar.gz ls -lh ~/open-webui.tar.gz # around 1-2 GB # Copy to USB cp ~/open-webui.tar.gz /media/usb/
bash — on SANDBOX server: load & run
# Load Docker image from USB docker load -i /mnt/usb/open-webui.tar.gz # Run Open WebUI — connects to local Ollama docker run -d \ --name open-webui \ --restart always \ -p 3000:8080 \ -v open-webui:/app/backend/data \ --add-host=host.docker.internal:host-gateway \ -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \ ghcr.io/open-webui/open-webui:main # Check it started docker ps docker logs open-webui --tail 20 # Access from browser on ANY PC on your network: # http://YOUR-SERVER-IP:3000 hostname -I | awk '{print "Access at: http://"$1":3000"}'
2
Option B — pip install (if no Docker, lightweight)
bash — pip install method (needs Python 3.11+)
# Install Python deps (download wheel files on internet machine first) # On internet machine: pip download open-webui -d ~/webui-wheels/ cp -r ~/webui-wheels/ /media/usb/ # On sandbox server — install from local wheels pip install open-webui \ --no-index \ --find-links=/mnt/usb/webui-wheels/ # Run WebUI DATA_DIR=~/.webui \ OLLAMA_BASE_URL=http://localhost:11434 \ open-webui serve --port 3000 & # Make it run on boot sudo tee /etc/systemd/system/open-webui.service <<EOF [Unit] Description=Open WebUI After=ollama.service [Service] ExecStart=/usr/local/bin/open-webui serve --port 3000 Environment="DATA_DIR=/home/$USER/.webui" Environment="OLLAMA_BASE_URL=http://localhost:11434" Restart=always User=$USER [Install] WantedBy=multi-user.target EOF sudo systemctl enable open-webui sudo systemctl start open-webui

Phase 4 — Add Terminal CLI Wrapper
Ask AI directly from your terminal — type: ai "how do I check memory?"
1
Install the 'ai' terminal command — one file, no dependencies
bash — create /usr/local/bin/ai
sudo tee /usr/local/bin/ai <<'SCRIPT' #!/bin/bash # ── Offline Linux Admin AI ────────────────────────────────── # Usage: ai "your question" # Usage: ai -m mistral:7b "your question" (choose model) # Usage: echo "long text" | ai "summarize this" MODEL="linux-admin:latest" OLLAMA="http://localhost:11434" # Allow model override with -m flag if [[ "$1" == "-m" ]]; then MODEL="$2" shift 2 fi if [[ -z "$1" ]]; then echo "Usage: ai [-m model] 'your linux question'" echo " pipe input: cat logfile | ai 'what errors are here?'" echo "" echo "Available models:" ollama list exit 1 fi # Check if piped input exists PIPED="" if ! tty -s; then PIPED=$(cat) fi PROMPT="$1" if [[ -n "$PIPED" ]]; then PROMPT="$1\n\nContext/Input:\n$PIPED" fi # Call Ollama API — stream=false for clean output RESPONSE=$(curl -s "$OLLAMA/api/generate" \ -H 'Content-Type: application/json' \ -d "{\"model\":\"$MODEL\",\"prompt\":\"$PROMPT\",\"stream\":false}") # Extract just the response text echo "$RESPONSE" | python3 -c \ "import sys,json; d=json.load(sys.stdin); print(d.get('response','Error: '+str(d)))" SCRIPT sudo chmod +x /usr/local/bin/ai echo "Done! Test with: ai 'how do I check disk usage?'"

Optimization Tips for Low-RAM / Low-CPU
Squeeze maximum performance from minimal hardware
★
All performance tuning commands in one place
bash — performance tuning for minimal hardware
## ── 1. USE SMALLER CONTEXT WINDOW ──────────────────────────── # Smaller context = less RAM used per request # Edit your Modelfile: PARAMETER num_ctx 2048 (instead of 4096) ollama create linux-admin-lite -f ~/LinuxAdminModelfile # rebuild after edit ## ── 2. USE AGGRESSIVE QUANTIZATION ─────────────────────────── # q4_0 = smallest, fastest, slightly less accurate ollama pull mistral:7b-instruct-q4_0 # 3.8 GB vs 4.1 GB default ## ── 3. SET CPU THREAD COUNT ─────────────────────────────────── # Set to actual core count (check with: nproc) echo 'Environment="OLLAMA_NUM_THREAD=4"' >> /etc/systemd/system/ollama.service ## ── 4. ENABLE SWAP (emergency if OOM) ──────────────────────── # Adds 8GB swap as safety net (slow but prevents crashes) sudo fallocate -l 8G /swapfile sudo chmod 600 /swapfile sudo mkswap /swapfile sudo swapon /swapfile echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab ## ── 5. KILL UNUSED SERVICES ─────────────────────────────────── # Free RAM by disabling services you don't need on sandbox sudo systemctl stop snapd cups bluetooth ModemManager avahi-daemon sudo systemctl disable snapd cups bluetooth ModemManager avahi-daemon ## ── 6. MONITOR RESOURCE USE WHILE RUNNING ───────────────────── watch -n 1 'free -h; echo "---"; ollama ps; echo "---"; top -bn1 | head -15' ## ── 7. CHECK HOW MUCH RAM MODEL IS USING ────────────────────── ollama ps # shows loaded models + RAM/VRAM usage ## ── 8. UNLOAD MODEL MANUALLY TO FREE RAM ────────────────────── ollama stop linux-admin # frees RAM immediately ## ── 9. SWAPPINESS TUNING ────────────────────────────────────── # Prefer RAM over swap for AI (reduces slowdowns) sudo sysctl -w vm.swappiness=10 echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.conf
